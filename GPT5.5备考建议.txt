**总体备考思路**
你不是准备学术考试，而是准备“个人研发能力考核”。所以每个知识点都要能做到三层：**讲清概念、判断适用场景、落到工程方案**。公司大概率不会只问“什么是 Transformer”，而会问“为什么你的项目要用 Transformer/RAG/LoRA/Agent，出了问题怎么定位”。

## 考点1：机器学习基础概念

**需要掌握**
- 监督学习、无监督学习、强化学习的区别：标签、目标函数、反馈信号不同。
- 分类、回归、聚类、降维、异常检测、推荐、排序等典型任务。
- 常用指标：Accuracy、Precision、Recall、F1、ROC-AUC、PR-AUC、MSE、MAE、RMSE、MAPE、NDCG、HitRate。
- 欠拟合/过拟合：训练集与验证集表现差异。
- 偏差-方差权衡：模型太简单偏差高，模型太复杂方差高。
- 常见模型：线性回归、逻辑回归、SVM、决策树、随机森林、GBDT/XGBoost、KMeans、PCA、朴素贝叶斯。
- 数据泄漏、样本不均衡、交叉验证、训练/验证/测试集划分。

**模拟题1**
二分类欺诈检测任务中，负样本占 99%，模型 Accuracy 达到 99%，是否说明模型很好？

**答案分析**
不一定。极度类别不均衡时，模型全部预测为负样本也可能有 99% Accuracy。应重点看 Recall、Precision、F1、PR-AUC、混淆矩阵，并结合业务代价设置阈值。如果漏检欺诈代价高，应优先提高 Recall；如果误杀正常用户代价高，应控制 Precision 和误报率。

**模拟题2**
训练集效果很好，验证集效果很差，如何处理？

**答案分析**
这是典型过拟合。可用更多数据、数据增强、正则化、Dropout、早停、降低模型复杂度、特征选择、交叉验证、检查数据泄漏。大模型场景下还要注意训练数据和评测数据是否污染。

## 考点2：深度学习网络结构

**需要掌握**
- CNN：卷积核、局部感受野、参数共享、池化、平移不变性，适合图像/局部模式。
- RNN/LSTM/GRU：序列建模，存在梯度消失、难并行、长依赖弱等问题。
- Transformer：Self-Attention、Q/K/V、位置编码、FFN、残差连接、LayerNorm。
- Attention 复杂度：标准注意力通常是 `O(n^2)`，长上下文需要稀疏注意力、滑窗、FlashAttention、KV cache 等优化。
- CNN/RNN/Transformer 的差异：CNN 偏局部，RNN 偏顺序递归，Transformer 偏全局依赖和并行计算。
- 图像、文本、语音、时序、多模态任务中的结构选择。

**模拟题1**
为什么 Transformer 比 RNN 更适合大规模语言模型？

**答案分析**
RNN 按时间步递归计算，难以并行，长距离依赖容易衰减。Transformer 通过 self-attention 直接建模任意 token 间关系，训练并行度高，更适合大规模预训练。但代价是上下文长度增加时注意力计算和显存成本上升。

**模拟题2**
CNN 在图像任务中有什么优势？为什么视频任务不能只用普通 CNN？

**答案分析**
CNN 能利用图像局部空间结构，参数共享降低复杂度。但视频不仅有空间关系，还有时间维度、运动、遮挡、跨帧一致性。视频任务通常需要 3D CNN、时序注意力、光流、Transformer 或时空联合建模。

## 考点3：大语言模型核心原理

**需要掌握**
- 架构类别：Encoder-only，如 BERT；Decoder-only，如 GPT/LLaMA；Encoder-Decoder，如 T5；MoE，如 Mixtral 类结构。
- 预训练：next token prediction、masked language modeling、多任务预训练。
- 微调：SFT、LoRA/QLoRA、Adapter、Prefix/Prompt Tuning、全参微调。
- 对齐：RLHF、RLAIF、DPO、IPO、KTO、reward model、preference data。
- PEFT 推理开销：LoRA 可合并权重，推理开销小；Adapter/Prefix 往往增加额外模块或 token，可能增加推理成本。
- 推理增强：CoT、Self-Consistency、Tree-of-Thought、ReAct、工具调用。
- 多模态扩展：视觉编码器 + projector + LLM；cross-attention；统一 token 化；多模态 instruction tuning。
- 大模型常见问题：幻觉、上下文遗忘、越狱、安全、长上下文退化、知识过期。

**模拟题1**
LoRA 和全参微调有什么区别？什么时候选 LoRA？

**答案分析**
全参微调更新所有参数，效果上限高，但显存、存储和训练成本大，也更容易灾难性遗忘。LoRA 冻结原模型，只训练低秩增量矩阵，成本低、易多任务切换，适合领域适配、风格适配和资源有限场景。若任务和基础模型差距极大，或需要深度能力改变，全参微调可能更合适。

**模拟题2**
DPO 和 RLHF 的核心区别是什么？

**答案分析**
RLHF 通常先训练 reward model，再用 PPO 等强化学习方法优化模型，流程复杂且不稳定。DPO 直接用偏好对数据优化策略模型，把“偏好样本中 chosen 应高于 rejected”转化为监督式目标，训练更简单稳定，常用于大模型对齐。

## 考点4：NLP 与 CV 任务体系

**需要掌握**
- 文本预处理：清洗、分词、tokenization、BPE/SentencePiece、停用词、归一化。
- 文本表示：one-hot、TF-IDF、Word2Vec、BERT embedding、LLM embedding。
- NLP 任务：分类、NER、关系抽取、问答、摘要、翻译、信息抽取、文本生成、检索、排序。
- RAG 流程：文档解析、切分、embedding、向量检索、rerank、上下文拼接、生成、引用与评估。
- 视觉任务：分类、检测、分割、OCR、姿态估计、图像生成、视频理解。
- 生成式视觉模型：GAN、VAE、Diffusion、Autoregressive、Flow Matching。
- 视频挑战：时序一致性、运动建模、主体保持、遮挡、长视频漂移、物理合理性。

**模拟题1**
RAG 系统回答错误，可能有哪些原因？

**答案分析**
可能是文档切分不合理、embedding 不匹配、召回失败、rerank 不准、上下文过长导致模型忽略关键信息、提示词约束不足、原始知识库错误、模型幻觉。优化方向包括改 chunk 策略、混合检索、重排序、query rewrite、答案引用、拒答机制和离线评测集。

**模拟题2**
视频理解相比图像理解多了哪些难点？

**答案分析**
视频有时间维度，需要理解动作、事件发展、跨帧目标一致性、镜头变化、遮挡恢复和因果关系。图像只处理单帧空间信息，视频还要处理时空联合建模，计算量和标注成本也更高。

## 建议补充考点5：大模型应用开发与评测

**需要掌握**
- Prompt Engineering：角色、任务、约束、示例、输出格式、反思/校验。
- Function Calling / Tool Calling：JSON schema、参数校验、工具路由、错误重试。
- 结构化输出：JSON mode、Pydantic 校验、解析失败修复。
- 大模型评测：准确性、鲁棒性、幻觉率、拒答率、延迟、成本、人工偏好。
- 应用架构：RAG、Agent、微调、缓存、日志、灰度发布、监控。
- 安全：提示词注入、数据泄漏、权限控制、审计。

**模拟题**
你要做一个企业知识库问答系统，如何设计评测集？

**答案分析**
应覆盖事实问答、复杂多跳问题、无答案问题、权限敏感问题、时效问题、表格/图片/长文档问题。指标包括召回率、答案准确率、引用正确率、幻觉率、拒答正确率、延迟和成本。最好有人工标注 golden answer，并区分检索错误和生成错误。

## 考点6：模型训练与优化技术

**需要掌握**
- 损失函数：MSE、MAE、Cross Entropy、Focal Loss、Contrastive Loss、Triplet Loss、Ranking Loss、Policy Loss。
- 优化器：SGD、Momentum、Adam、AdamW、Adafactor。
- 学习率策略：warmup、cosine decay、step decay、linear decay。
- 正则化：L1/L2、Dropout、Label Smoothing、数据增强、早停。
- 大规模训练：混合精度、梯度累积、梯度裁剪、数据并行、张量并行、流水线并行、ZeRO、checkpointing。
- 超参搜索：grid search、random search、Bayesian optimization、early stopping。

**模拟题**
训练大模型时 loss 突然爆炸，如何排查？

**答案分析**
检查学习率是否过大、梯度是否爆炸、数据是否异常、label 是否错位、混合精度是否溢出、loss mask 是否错误、batch 中是否有极端样本。处理方式包括降低学习率、加 gradient clipping、检查数据管道、开启 loss scale、缩小 batch 做复现。

## 考点7：模型部署与工程化实践

**需要掌握**
- 轻量化：量化 INT8/INT4、剪枝、蒸馏、低秩分解、MoE 路由优化。
- 推理优化：KV cache、continuous batching、prefix cache、speculative decoding、FlashAttention。
- 推理框架：vLLM、TensorRT-LLM、Triton、ONNX Runtime、llama.cpp。
- 服务架构：API 网关、队列、负载均衡、限流、熔断、监控、日志。
- 云端 vs 边缘端：云端算力强、易扩展；边缘端低延迟、隐私好，但资源有限。
- 指标：吞吐、延迟、显存、QPS、首 token 延迟、每 token 延迟、成本。

**模拟题**
一个 LLM 服务延迟高、GPU 利用率低，如何优化？

**答案分析**
可以使用 batch/continuous batching 提高吞吐，开启 KV cache 和 prefix cache，量化降低显存，使用 vLLM/TensorRT-LLM，优化 prompt 长度，做请求队列和动态批处理，热点答案缓存，必要时模型蒸馏或拆分大小模型路由。

## 考点8：AI 智能体与智能系统

**需要掌握**
- Agent 四模块：感知、规划、记忆、行动。
- 规划模式：ReAct、Plan-and-Execute、Reflection、Tree-of-Thought。
- 记忆：短期上下文、长期向量记忆、结构化数据库、事件日志。
- 工具调用：函数 schema、权限、参数校验、结果观察、失败重试。
- 多智能体：角色分工、协作、辩论、投票、任务分解。
- 标准协议：MCP 等外部工具/数据源接入协议。
- 框架：LangChain、LlamaIndex、AutoGen、CrewAI、Semantic Kernel。
- 工程风险：死循环、错误工具调用、权限越界、幻觉计划、成本失控。

**模拟题**
设计一个电力设备巡检 Agent，需要哪些模块？

**答案分析**
感知模块接入传感器、图像、工单、历史检修记录；规划模块拆解巡检任务；记忆模块保存设备历史故障和维修记录；工具模块调用数据库、图像识别模型、告警系统和工单系统；行动模块生成诊断建议或派单。还要加入权限控制、人工确认、日志审计和异常回退。

## 考点9：数据工程与特征处理

**需要掌握**
- 数据清洗：重复值、缺失值、异常值、格式统一、噪声处理。
- 缺失处理：删除、均值/中位数填充、模型填充、时间序列前向填充。
- 特征工程：交叉特征、统计特征、时间窗口特征、类别编码、离散化。
- 归一化/标准化：MinMax、Z-score、RobustScaler。
- 降维与特征选择：PCA、LDA、相关性筛选、L1、树模型重要性。
- 时序处理：滑动窗口、滞后特征、周期特征、节假日特征、泄漏防控。
- 向量检索：embedding、ANN、HNSW、IVF、PQ、召回与重排。
- 数据管道：ETL、数据校验、feature store、离线/在线一致性。

**模拟题**
做负荷预测时，为什么不能随机划分训练集和测试集？

**答案分析**
负荷预测是时间序列任务，随机划分会把未来信息泄漏到训练集中，导致评估虚高。应按时间顺序划分训练、验证、测试集，并注意构造特征时不能使用预测时刻之后的数据。

## 考点10：算法基础、AI伦理与电力行业应用

**需要掌握**
- 排序：冒泡、插入、归并、快排、堆排，复杂度和稳定性。
- 查找：二分查找、哈希表、树、堆。
- 动态规划：最优子结构、重叠子问题，适合全局最优。
- 贪心算法：局部最优可推出全局最优时适用。
- 图算法：BFS、DFS、Dijkstra、最小生成树，可用于电网拓扑分析。
- AI 伦理：鲁棒性、公平性、可解释性、隐私保护、安全合规。
- 电力场景：负荷预测、设备故障诊断、用户画像、推荐、调度优化、数字孪生、智能客服、巡检 Agent。
- 电力行业重点：稳定性、安全性、可解释性、实时性、合规审计。

**模拟题1**
动态规划和贪心有什么区别？

**答案分析**
动态规划保存子问题结果，通过状态转移求全局最优，适合存在重叠子问题和最优子结构的任务。贪心每一步选择当前最优，只有在局部最优能保证全局最优时成立。调度优化、路径规划中二者都常见，但需要证明贪心选择性质。

**模拟题2**
AI 用于电网调度优化时，为什么不能只追求模型效果？

**答案分析**
电网是高安全、高稳定系统。除了预测或优化效果，还要考虑约束满足、可解释性、鲁棒性、极端场景、安全边界、人工接管、审计记录和合规要求。模型建议必须能被验证，不能黑箱直接控制关键设备。
